### 1. ID System (Critical!)
```

  â€¢ Assigned at creation
  â€¢ Never changes
  â€¢ Use for: Record identification in Tally
  â€¢ Update: Keep for reference

ALTERID
  â€¢ Global modification counter
  â€¢ Increments on ANY change in Tally
  â€¢ Use for: Incremental sync (WHERE ALTERID > lastSync)
  â€¢ Critical for: Change detection

GUID
  â€¢ Globally unique identifier
  â€¢ Unique per record
  â€¢ Use for: Deduplication, sync mapping
  â€¢ Benefit: Works across systems
```

### 3. Sync Strategy
```
Incremental Sync (Most Efficient):
  1. Get last synced ALTERID from database
  2. Query: WHERE ALTERID > lastAlterID
  3. Process returned records
  4. Update local lastAlterID
  5. Repeat on schedule (e.g., every 5 seconds)

Full Historical Sync (Initial Setup):
  1. Start with ALTERID = 0
  2. Fetch in chunks (LIMIT 1000)
  3. Track progress
  4. Store all records
  5. Resume capability if interrupted

Conflict Resolution:
  â€¢ Tally is source of truth
  â€¢ Always prefer newer ALTERID
  â€¢ Match by GUID if same ALTERID
  â€¢ Mark deleted records locally
```
